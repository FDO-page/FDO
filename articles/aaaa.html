创建你的个人网站：免费且简单的教程

发布日期：2025-01-22 | 作者：FDO 技术
>返回文章列表

想要拥有一个展示个人文章、作品或作为个人主页的网站作为FDO的分支网站吗？静态网站是一个不错的选择，它不仅搭建简单，而且完全免费。本文将引导你通过已有的网站框架，快速搭建一个FDO的分支网站（当然也可以用于属于你自己的展示型网站）。

准备工作

在开始之前，你需要做好以下准备：

  1. GitHub 账号: 这是构建网站的基础平台。
    • 推荐使用 GitHub 网页版，而非客户端，以确保所有功能可用。
    • GitHub 是一个国外平台，无需担心内容审查，但连接速度可能不稳定。建议在网络条件允许的情况下使用，或考虑开启代理。
    • 可以使用任何邮箱注册，但为了安全起见，建议避免使用国内邮箱。如果你有使用代理，可以先注册一个 Proton 邮箱，再用它来注册 GitHub。
    • 注意： 你在 GitHub 上设置的用户名会影响你最终的网站地址，所以尽量选择一个好记的用户名。
  2. 浏览器: 建议同时打开本教程页面和 GitHub 页面，并确保 GitHub 账号已登录。

选择网站风格

首先，选择你喜欢的网站风格。我们提供了两种风格的模板：

  • 蓝绿色毛玻璃效果:
    • 主页模板
    • 文章页模板
  • 简洁黑白风格:
    • 主页模板
    • 文章页模板

请分别打开你选择的主页和文章页模板链接，以便后续复制代码。

创建 GitHub 仓库

  1. 返回 GitHub 主页，确保你已登录。
  2. 点击 "Create a new repository" 按钮（绿色的按钮）。
  3. 在 "name your repository" 输入框中，输入你的仓库名称。
    • 注意： 仓库名称也会影响你最终的网站地址，所以尽量选择好记的名称。
    • 仓库名称的格式通常是：你的用户名.github.io，例如 example.github.io。
  4. 将仓库设置为 "Public"，因为免费的 GitHub Actions 在私有仓库中有使用限制。
  5. 点击 "Create repository" 创建仓库。

创建 HTML 文件

  1. 进入你创建的仓库页面。此时仓库是空的，会出现一些提示信息，不用担心。
  2. 找到 浅蓝色面板 中的 “creating a new file” (通常是第一条蓝色文字链接)，点击它。
  3. 在 “name your file” 输入框中，输入 index.html。
    • index.html 是网站的默认首页文件名，必须使用这个名称。
  4. 复制 你之前选择的主页模板的代码，并 粘贴 到代码编辑框中。
  5. 点击右上方绿色的 "Commit new file" 按钮 保存文件。
  6. 在下一个页面，再次点击绿色的 “Commit new file” 按钮。
    • 这个页面是填写更新日志的，可选择不填写直接保存。
  7. 如果成功跳转到仓库主页，说明文件已成功保存。即使有提示信息，也可以忽略。

创建文章页面

重复以上步骤，创建第二个仓库和 index.html 文件，这次 粘贴文章页模板的代码。

  • 注意： 仓库名称不可重复，但不同仓库中的 index.html 文件名可以重复。

部署 GitHub Pages

  1. 打开一个仓库，点击右上角的省略号 (电脑端不用) ，再点击 “Settings” 按钮。
  2. 在左侧导航栏中，找到 “Pages” 并点击。
    • 手机端可能需要向下滑动才能看到 “Pages” 选项。
  3. 在 “Source” 部分，找到 “None” 按钮并点击，选择 “main” 分支。
    • 表示将你的网站文件部署在 main 分支下。如果创建了其他分支，也可以选择其他分支。
  4. 点击右边的 "Save" 按钮 保存设置。
  5. 刷新几次页面，稍等片刻 (通常一分钟左右)，你会看到一个以 "Your site is live at" 开头的链接，这就是你的网站地址。
  6. 点击链接右边的 “Visit site” 按钮，检查你的网站是否正常显示。
  7. 重复以上步骤， 将另一个仓库也开启 GitHub Pages 功能。

自定义网站内容

现在你已经有了自己的网站，可以通过以下方式修改内容：

  1. 进入仓库代码页面，点击 index.html 文件名。
  2. 点击右上角的 铅笔图标 (手机端是省略号，选择 “Edit this file”)。
  3. 滑动页面，找到以下内容，你可以修改它们：
    • 第一段中文文本: 这里是网页名称， 前面的内容是网页名称，显示在浏览器标签页的标题栏。
    • 第二段中文文本:

      这里是文章标题

       ， 前面的内容是文章标题，显示在网页的大字标题部分。
    • 第三段中文文本:

      这里是正文内容

      ，

      前面的内容是文章的正文内容。

代码说明

  • 引用链接: 使用 [link:显示文字](链接地址) 格式，如 [link:我的博客](https://example.com)。
  • 引用图片: 使用 [image:图片链接] 格式，如 [image:https://example.com/image.jpg]。
    • 需要使用图床来获取图片链接，推荐使用 sm.ms，免费且国内访问速度较快。
    • 注意: 确保图片链接以图片格式（如 .jpg, .png, .gif）结尾，如果不是，请检查你的图片链接是否正确。
  • 换行: 在正文中使用 \n 来换行。
  • 文章格式转换: 如果觉得手动换行麻烦，可以使用 这个网站 来转换你的文章格式。

使用 Cloudflare 加速访问（可选）

如果你的网站访问不稳定，可以考虑使用 Cloudflare 加速访问。对于静态网站，Cloudflare 提供免费的部署服务。

Cloudflare 部署步骤

  1. 使用你注册 GitHub 的邮箱注册并登录 Cloudflare 账号。（也可以使用其他邮箱，但关联 GitHub 账号会麻烦一些）
  2. 下滑找到 “start building” 按钮并点击。
  3. 选择第一个 “Build” 开头的选项。
  4. Cloudflare 会检查你的邮箱，打开邮箱找到验证邮件，并点击其中的链接。
  5. 点击 “Pages” 选项。
  6. 点击 “Connect to git” 按钮。
  7. 在跳转的页面中，点击 “Connect GitHub”。
  8. 在接下来的页面中，直接下滑并点击绿色的 “Install” 按钮。
  9. 登录你的 GitHub 账号。
  10. 选择你要部署的仓库，并点击。
  11. 输入项目名称，注意，这个名称也会影响你的 Cloudflare 免费网址，尽量选择简短好记且和 Cloudflare 提供的网址不同的名称。
  12. 滑动到页面底部，点击蓝色的 "Save" 按钮。
  13. 等待部署完成后，点击右下角蓝色按钮退出，你会看到你部署的网站，点击右上方的 “Visit Site” 按钮进行检查。
  14. 如果有其他 GitHub 仓库需要部署，点击左上角的 "Workers" 重复以上步骤。

创建分支网站（子目录）

如果你想创建类似于 joyance.page/cn/A 这样的分支网站，可以使用以下步骤：

  1. 在你的 GitHub 仓库中创建一个新的文件夹， 例如 cn。
  2. 在 cn 文件夹中，创建另一个文件夹，例如 A。
  3. 在 A 文件夹中，创建一个 index.html 文件，并将你想要展示的内容（如文章页模板代码）粘贴进去。
    • 这个 index.html 文件就是你的分支网站页面。
    • 你可以根据需要，在 cn 文件夹下创建多个子文件夹 (如 B, C, ...) 来组织不同的分支网站内容。
  4. 提交 (Commit) 这些更改 到你的 GitHub 仓库。
  5. 确保你的 GitHub Pages 设置 指向 main 分支（或者你实际使用的分支）。
  6. 稍等片刻，让 GitHub Pages 部署新的更改。
  7. 访问你的分支网站，地址为 你的用户名.github.io/cn/A。

注意事项:

  • 目录结构： 你的用户名.github.io/cn/A 中， cn 是你的一个文件夹，A 是 cn 下的一个文件夹， index.html 放置在 A 文件夹下。
  • 多级目录： 你可以在 A 文件夹下继续创建多级目录和 index.html 文件，来组织更复杂的内容。例如 你的用户名.github.io/cn/A/subpage。
  • 资源链接： 如果你在分支网站中引用图片或其他资源，确保你的链接是相对于当前页面的路径，或者使用绝对路径。例如，如果你在 A/index.html 中要引用同目录下的图片 image.jpg，可以使用 。
  • Cloudflare 支持: 如果你使用 Cloudflare 加速，以上的分支网站也会被正确部署，访问路径与 GitHub Pages 相同。

总结

通过以上步骤，你就拥有了自己的个人网站和分支网站，快去尝试构建吧！
